이 앱의 핵심 타겟인 한국인 사용자들에게 **'인기 유튜버의 검증 영상'**은 미슐랭 가이드만큼이나 강력한 신뢰 지표(Social Proof)입니다. 이를 앱 내에서 **'로우 데이터(가공되지 않은 원본 영상)'** 형태로 보여주기 위한 기술적 **후킹(Hooking/Fetching)** 및 통합 전략을 소스에 기반하여 설계해 드립니다.

단순히 영상을 검색해서 보여주는 것을 넘어, **"긴 영상 속에서 해당 식당이 나오는 정확한 시점(Timeline)"**을 찾아내어 매핑하는 것이 이 기능의 핵심 경쟁력입니다.

### 🏛️ 유튜브 데이터 후킹 및 통합 파이프라인

소스,,에 명시된 `Source Fetcher`와 `Gemini Processing` 모듈을 활용하여 다음 4단계 프로세스로 구현합니다.

#### 1단계: 타겟 유튜버 리스트 확보 (White-listing)
모든 유튜버의 영상을 긁어오는 것은 비효율적이며 데이터 품질을 떨어뜨립니다. 신뢰도 높은 **'검증된 크리에이터'**를 DB화하는 것이 먼저입니다.

*   **대상 선정:**
    *   **메가 인플루언서:** 성시경(먹을텐데), 백종원(님아 그 시장을 가오), 최자로드 등.
    *   **여행 전문:** 빠니보틀, 곽튜브, 혹은 소스,에서 언급된 '파리 외노자(Monsieur Oh)', 'CHUNG Haemi' 같은 현지 특화 유튜버.
*   **구현 방법:** 관리자 페이지에서 '채널 ID'를 등록하여 해당 채널의 영상만 주기적으로 트래킹(Tracking)합니다.

#### 2단계: API 기반 데이터 수집 (Fetching)
유튜브의 정책을 준수하면서 안정적으로 데이터를 가져오기 위해 **YouTube Data API v3**를 사용합니다.

*   **API Query:** `GET search` (채널 ID + "맛집" or "여행" 키워드).
*   **가져올 데이터:**
    *   `Video ID`: 영상 고유 주소.
    *   `Snippet`: 제목, 설명, 썸네일.
    *   `Captions` (핵심): **영상 자막(Transcript)** 데이터. (이것이 후킹의 핵심입니다.)

#### 3단계: AI를 활용한 '타임스탬프' 매핑 (The Intelligence)
보통 유튜브 영상은 10~20분 길이이며 여러 장소가 나옵니다. 사용자가 영상을 처음부터 보게 하면 이탈합니다. **Gemini Text API**를 사용하여 식당이 등장하는 **정확한 시간(초)**을 추출해야 합니다.

*   **AI 프롬프트 설계 (예시):**
    > **Input:** 영상 자막 스크립트 전문 + 우리 앱의 POI(장소) 리스트.
    > **Request:** "이 자막에서 '삼부자' 식당에 대해 이야기하기 시작하는 정확한 시작 시간(Start Time)과 끝나는 시간(End Time)을 추출하고, 그 내용을 3줄로 요약해줘."
*   **결과 데이터:**
    *   `Place_ID`: "Sambuja_Paris"
    *   `Video_URL`: `youtube.com/watch?v=xyz&t=345s` (345초부터 시작하는 링크 생성)
    *   `Review_Summary`: "김치찌개가 한국보다 깊은 맛이라고 극찬함."

#### 4단계: 앱 내 표출 (UI/UX Integration)
저작권 문제를 피하고 앱 용량을 줄이기 위해 영상을 직접 다운로드하지 않고 **임베디드 플레이어(Embedded Player)**를 사용합니다.

*   **숏폼 스타일 프리뷰:** 소스,의 '리얼 숏폼 미리보기' 기능을 응용합니다.
    *   앱에서 식당을 클릭하면, 상단에 유튜버의 영상이 **자동으로 해당 구간(345초)부터 재생**되도록 설정합니다.
    *   **UI 구성:**
        *   **[유튜버 Pick]** 뱃지 부착.
        *   영상 하단에 "성시경이 '인생 국물'이라고 극찬한 순간" 같은 AI 요약 코멘트 노출.

### 🧪 기술적 구현을 위한 N8N 워크플로우 예시

개발팀에 전달할 구체적인 로직입니다 (소스, 참고).

1.  **[Trigger Node]** 매일 자정, 등록된 '유튜버 채널 리스트' 업데이트 확인.
2.  **[YouTube API Node]** 신규 업로드된 영상의 자막(Transcript) 다운로드.
3.  **[Gemini AI Node]** 자막 텍스트 분석:
    *   *Task:* 자막 내에 등장하는 '식당 이름'을 추출하고, 우리 앱 DB의 장소명과 매칭(Fuzzy Matching).
    *   *Output:* `{ "Place": "Kwon", "Timestamp": "12m 30s", "Sentiment": "Positive" }`
4.  **[DB Update Node]** 해당 식당 정보에 `Influencer_Video_Link` 필드 업데이트.

### ⚠️ 주의사항 및 팁

1.  **저작권 (Copyright):** 영상을 다운로드해서 앱 서버에 올리면 저작권 위반입니다. 반드시 **YouTube IFrame Player API**를 사용하여 앱 내에서 유튜브가 재생되도록 해야 합니다. 이는 합법적이며 유튜버의 조회수도 올려주므로 상생 구조가 됩니다.
2.  **데이터 정확도:** 유튜버가 식당 이름을 말하지 않고 "여기", "이 집"이라고 하는 경우 AI가 놓칠 수 있습니다. 이 경우 영상 설명란의 링크나 댓글 고정핀 정보를 AI가 추가로 크롤링하도록(Source) 보완해야 합니다.
3.  **현지 유튜버 활용:** 소스의 '마 키친(Ma Kitchen)' 리뷰 영상(유튜버 CHUNG Haemi)이나의 '파리연남자' 같은 현지 거주 유튜버의 영상은 '현지 검증(Local Buzz)' 점수에 가중치를 더 높게 부여하십시오.

이 방식을 통해 귀하의 앱은 단순한 정보 나열이 아닌, **"유명 유튜버가 검증한 구간만 쏙쏙 골라 보여주는"** 강력한 큐레이션 기능을 갖추게 될 것입니다.
